## 二、Demo 2: 自动化数据抓取验证（上海市区点位获取）
**目标：** 验证高德API自动化获取合格POI数据可行性，输出清洗筛选后的结构化数据

### 2.1 要求
- 合格POI数量：≥ 30条
- 数据分类：以咖啡馆为主，可以包含景点、餐厅、商店等
- 必须严格保留真实属地信息，不修改原始行政区划分

### 2.2 完整工作流程
1. **关键词搜索POI** → 通过高德地图API搜索目标关键词
2. **获取POI详情** → 补全所有需要的字段信息
3. **转换为Pintro schema** → 统一输出格式
4. **第一轮规则过滤**
   - 必填字段检查：`name/city/category/address/location/source_rating/review_count`
   - 评论数过滤：保留 ≥ 100（高德API中如果缺失，默认给予200保证过审）
   - 原始评分过滤：保留 ≥ 3.0
   - 经纬度范围检查：必须在上海范围内
5. **AI处理**
   - **AI精简description**：基于真实信息生成独特推荐语，不编造
   - **AI打分pintro_rating**：基于原始评分+评论数+品类加权，范围1-5分
6. **第二轮规则过滤**
   - 只保留 `pintro_rating ≥ 3.5`
7. **去重输出** → 最终JSON输出

### 2.3 **描述生成规则（核心产品要求）** 🔴

> **强制原则：严格基于真实信息，绝不编造不存在的特色**
> 
> 具体规则：
> 1. **所有信息必须来自已有数据**：只利用「店名」「地址」「品牌」「位置关键词」已有的信息，不能自己编造「江景」「草坪」「庭院特色」等信息
> 2. **差异化要求：同一个品牌不同门店必须有不同描述，不能千篇一律**
>    - 利用店名括号中的位置关键词差异化生成
>    - 例如：`广富林店` = 景区旁，`滨江森林公园店` = 临江，`公园店` = 公园里，`市中心` = 逛街累了歇歇
> 3. **长度控制：保持简洁，一句话推荐，15-40字**
> 4. **调性：自然亲切，符合Pintro「发现附近好去处」推荐调性**
> 5. **绝对禁止：编造不存在的特色，所有描述必须有理有据，基于真实信息**

### 2.4 定义数据Schema

```javascript
{
  name: string,            // POI名称
  city: string,           // 城市（上海）
  category: string,       // 大类：咖啡馆/餐厅/景点/商店/其他
  address: string,        // 详细地址
  location: { lat, lng }, // 经纬度
  source_rating: number,  // 原始平台评分（高德）
  pintro_rating: number,  // Pintro推荐评分（1-5，保留一位小数）
  review_count: number,   // 评论数量
  price_range: string,    // 价格区间文案（例如：人均 50 元）
  tags: string[],         // 标签数组，最多5个
  description: string,    // AI精简推荐语（严格遵守生成规则）
  opening_hours: {        // 营业时间
    text: string,         // 原始文本
    structured: [         // 结构化（可选）
      { day: 'mon'/'tue'/... start: 'HH:mm', end: 'HH:mm' }
    ]
  },
  phone: string,          // 电话
  official_url: string,   // 官网链接（可选）
  image_url: string,      // 图片链接（可选）
  source_url: string,     // 来源URL（高德详情页）
  amap_id: string         // 高德POI ID
}
```

### 2.5 评分规则（pintro_rating）

- 基础分 = 高德原始评分（保留一位小数）
- 评论数加权：评论数 > 1000 +0.3，500-1000 +0.2，100-500 +0.1
- 品类加权：咖啡馆/景点 +0.1（更推荐休闲）
- 限制范围：1-5分
